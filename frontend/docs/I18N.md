# Vexora CRM — Internationalization

## Locales

| Code | Default | Storage key |
|------|---------|-------------|
| `ru` | Yes | `vexora.locale` in `localStorage` |
| `en` | | |
| `kk` | | |

## Architecture

- **Catalogs:** `src/i18n/messages/{en,ru,kk}.ts` — nested keys (`nav.deals`, `deals.viewAll`, …).
- **Runtime:** `translate()` / `t()` in `src/i18n/translate.ts`; `LocaleProvider` syncs React state + `setRuntimeLocale()` for non-React libs.
- **Hooks:** `useTranslation()` → `{ locale, setLocale, t }`.
- **UI switcher:** Profile menu → `LanguageSwitcherRow` (no page reload).

## Adding strings

1. Add key to `messages/en.ts`.
2. Mirror in `ru.ts` and `kk.ts`.
3. Use `const { t } = useTranslation()` in components, or `translate("key")` in libs.

## Deployment

- No env vars required for i18n.
- Rebuild frontend after message changes: `npm run build` in `frontend/`.
- Default locale is Russian until the user picks another language (persisted client-side).
- After deploy: switch **Русский** in profile menu and spot-check Deals, Tasks, client profile, Analytics, Settings/Team.
